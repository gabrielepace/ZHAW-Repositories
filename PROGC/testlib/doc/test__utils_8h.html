Boiler-plate code as a macro to define the complete body of the main function of the tests suite.

Parameters
[in]suiteThe name of the test suite (const char *).
[in]setupThe setup callback function which is executed before the first test is executed (int setup(void)).
[in]cleanupThe teardown callback function which is executed after the last test is executed (int teardown(void)).
[in]...The variable list of test_function callback (void test(void)) which constitue the test cases. They are executed in the given sequence.
Remarks
Example code (see also CUnit Framework Documentation):
// setup and teardown
int setup(void)
{
// do some initialization if needed
// ...
return 0; // success
}
int teardown(void)
{
// do some cleanup if needed
// ...
return 0; // success
}
// tests
void test_main_no_args(void)
{
// arrange
// ...
// act
// ...
// assert (use the CUnit CU_ASSERT_... macros)
// ...
}
void test_main_one_arg(void)
{
//...
}
void test_main_two_args(void)
{
//...
}
// execute the tests
int main(void)
{
TestMainBasic("Hello World", setup, teardown
, test_main_no_args
, test_main_one_arg
, test_main_two_args
);
}

◆ file_exists()

int file_exists ( const char  file_path[])

Gracefully checks by means of fopen(..., "r") if a file exists.

Parameters
[in]file_pathThe file to check for existence.
Returns
Returns 0 if the file does not exist, 1 otherwise.
Remarks
If the file does not exist, errno was affected. This function takes care of this, i.e. it safes and restores the original errno state.
In case of an error situation, the function fails with a hard assert.h assertion violation. This allows to use the function outside of a tests, e.g. in setup and teardown functions.
